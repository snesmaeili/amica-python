# amica-python validation — paper-relevant updates

This document summarizes the latest validation runs and bug fixes for the
amica-python paper. Use it as the reference for the Overleaf analysis section.

---

## 1. Bug fixes (commit `a7dc3f7`)

### 1.1 `fig_amica_specific` — `NameError` and shape broadcast
**Files:** `validation/quick_full_check.py`

The figure-generation function `fig_amica_specific` referenced an undefined
`ica_dict` and passed already-computed sources `(n_comp, n_samples)` to
`plot_parameter_summary`, which internally re-applies the whitener
`(n_comp, n_channels)` and the centering vector `(n_channels,)` — causing a
`(30,50000) vs (118,1)` broadcast error.

**Fix:** pass raw channel data `(n_channels, n_samples)` directly. The
visualization helper handles whitening and unmixing itself.

### 1.2 AMICA under-convergence — `MAX_ITER` was too low
**Files:** `validation/quick_full_check.py`, `validation/run_highdens_validation.py`

With `MAX_ITER=200`, AMICA was *under-trained* on ds004505:

| Method  | Brain ICs | MIR    | Iter | Time   |
|---------|-----------|--------|------|--------|
| AMICA   | **0**     | -14.1  | 200  | 13 s   |
| Picard  | 1         | -141.2 | 71   | 173 s  |
| Infomax | 1         | -141.4 | 134  | 184 s  |

The AMICA log-likelihood was still climbing at iter 200 (`converged=False`),
the mixing matrix was full rank, alpha summed to 1, but the mutual-information
reduction was a full order of magnitude worse than Picard/Infomax — meaning
the components were only weakly independent. AMICA's adaptive mixture
densities require many more iterations than fixed-density Infomax / Picard.

**Fix:** introduced `AMICA_MAX_ITER=2000` (matches MATLAB AMICA defaults).
Picard and Infomax early-stop well before 2000, so the bump only affects
AMICA. Result on ds004505 sub-01 (GPU, A100):

| Method  | Brain ICs | MIR    | Iter | Time   |
|---------|-----------|--------|------|--------|
| AMICA   | **1**     | ≈ -141 | 2000 | 72 s   |
| Picard  | 1         | -141.2 | 71   | 175 s  |
| Infomax | 1         | -141.4 | 134  | 151 s  |

AMICA now matches the other methods on brain-IC count and is the **fastest**
of the three on GPU (72 s for 2000 iter vs 151–175 s for the other two on CPU).

## 3. Full validation run (job array `58939673`)

Submitted on Narval as a 25-task GPU array (one task per subject).

**Submit script:** `validation/submit_full_validation_gpu.sh`
**Underlying script:** `validation/run_highdens_validation.py`

### Configuration

| Setting          | Value                                 |
|------------------|---------------------------------------|
| Dataset          | ds004505 (Frank 2025 high-density EEG)|
| Subjects         | sub-01 … sub-25 (job array 1–25)     |
| Methods          | AMICA, Picard, Infomax, FastICA       |
| HP filters       | 1.0 Hz, 2.0 Hz (sweep)                |
| Sampling rate    | 250 Hz                                |
| Min κ (kappa)    | 30                                    |
| n_components     | data-driven (rank-1, capped at 60)    |
| MAX_ITER         | 2000 (all methods; early-stop honored)|
| AMICA num_mix    | 3                                     |
| Random state     | 42                                    |
| Hardware         | 1× A100 GPU + 4 CPU + 32 GB RAM       |
| Wall-time / task | 2 h                                   |

### Metrics computed per (subject, hp, method)

- **ICLabel** counts (brain / muscle / eye / heart / line / channel / other),
  plus brain-IC counts at >50% and >70% confidence
- **Kurtosis-brain-like** count (super-Gaussian but not extreme: 0 < kurt < 10)
- **Alpha-peaked IC** count (PSD peak in 8–13 Hz)
- **MIR** (mutual information reduction relative to raw channels)
- **Reconstruction error**
- **Time** and **iteration count**

### Output

Per-subject per-HP JSON files:
```
validation/results/benchmark_sub-XX_hp{1.0,2.0}hz.json
```
A combined `benchmark_combined.json` is written by the per-subject script
(each task produces its own copy; merge across tasks post-hoc).

## 4. What to update in the Overleaf paper

1. **Methods section — AMICA configuration**
   Use `MAX_ITER=2000`, `num_mix_comps=3`, `random_state=42`. State that
   AMICA's adaptive-mixture optimization requires more iterations than
   fixed-density methods, but on GPU each iteration is so cheap that 2000
   iterations of AMICA still completes faster than ~70–135 iterations of
   Picard/Infomax on CPU.

2. **Numerical-parity table** — unchanged from `matlab_parity_results.json`
   (already in `validation/results/`).

3. **Figure 1 — multi-metric comparison**
   Source: `validation/results/figures/check_multimetric.{png,pdf}` (sub-01,
   illustrative) and the equivalent generated from `benchmark_combined.json`
   across all 25 subjects via `validation/generate_publication_figures.py`.

4. **Figure 2 — IC topomap grid**
   Source: `validation/results/figures/check_topoplots.{png,pdf}` for sub-01
   illustration; per-subject versions can be generated from the saved ICA
   pickles.

5. **Figure 3 — convergence curve**
   `validation/results/figures/check_convergence.{png,pdf}`. Highlight that
   AMICA still climbs slowly past iter 200 — this is the empirical evidence
   for `MAX_ITER ≥ 2000`.

6. **Figure 4 — AMICA parameter summary**
   `validation/results/figures/check_amica_params.{png,pdf}`. Shows the
   adaptive mixture (α, β, ρ) per IC — the signature feature of AMICA that
   no other ICA method has.

7. **Speed claim**
   On 1× A100, AMICA at 2000 iter (118 channels, 30 components, 1.2 M
   samples) = **72 s**. Picard / Infomax on CPU = **151–175 s**. AMICA is
   ~2× faster than Picard at iso-quality on GPU.

8. **Quality claim**
   On ds004505 sub-01 (quick check), AMICA recovers the same number of
   ICLabel-confident brain components as Picard and Infomax (1/30), with
   matching MIR (≈ −141 nats). For full quantitative claims, wait for the
   25-subject array job (`58939673`) to finish and aggregate
   `benchmark_sub-*_hp*.json`.
